Installing the MainWP Wordfence Extension requires two steps:

- Install and activate the add-on on your MainWP Dashboard site.
- Install and activate Wordfence Security on your child sites.

### 1\. Installing the Add-on

The MainWP Wordfence Extension is a WordPress plugin. Follow these steps on your MainWP Dashboard site, not on a child site:

<Steps>
<Step title="Go to WP Admin > Plugins > Add Plugin">
</Step>
<Step title="Click Upload Plugin">
</Step>
<Step title="Click Choose File">
</Step>
<Step title="Select mainwp-wordfence-extension.zip and click Install Now">
</Step>
<Step title="Click Activate Plugin after installation">
</Step>
<Step title="Go to MainWP > Add-ons > Manage Add-ons">
</Step>
<Step title="Locate Wordfence Integration and click Enable">
</Step>
</Steps>

The add-on is now ready for use. Alternatively, upload the extracted `mainwp-wordfence-extension` directory to `wp-content/plugins` on the MainWP Dashboard site, then activate and enable it.

### 2\. Installing Wordfence Security

Install and activate Wordfence Security on each child site that you want to manage through the extension.

<Steps>
<Step title="Go to WP Admin > Plugins > Add Plugin on the child site">
</Step>
<Step title="Search for Wordfence">
</Step>
<Step title="Locate Wordfence Security and click Install Now">

</Step>
<Step title="Sync the child site with your MainWP Dashboard">
</Step>
</Steps>

The MainWP Wordfence Extension can now detect and manage Wordfence Security on that child site.

## Wordfence Settings

### Global Settings

Go to **MainWP > Add-ons > Security > Wordfence**. The extension provides these network-level tabs:

- Wordfence Overview
- Options
- Live Traffic
- Blocking
- Firewall
- Scan
- Diagnostics

Use the settings tabs to define network defaults. Saving a network settings section synchronizes that section to child sites that are not using individual overrides. Sites with **Override general settings** enabled keep their individual settings.

If you install Wordfence Security on additional child sites, sync them and re-save each relevant network settings tab to apply the defaults.

### Individual Site Settings

The extension also provides per-site Wordfence settings.

## Wordfence Dashboard

Go to **MainWP > Add-ons > Security > Wordfence** and open the **Wordfence Overview** tab. The sites table shows child sites where Wordfence Security is installed, including plugin status, version, visibility, last scan, and update availability.

- If the plugin is inactive, the **Activate Plugin** action appears in the site row.
- If an update is available, the **Update Plugin** action appears in the site row.

### Available Actions

The row action menu includes **Overview**, **Edit**, **Open Wordfence**, and **Hide Plugin** or **Unhide Plugin**. For active installations, it also includes **Scan Site**, **Live Traffic**, **Blocking**, and **Scan Results**. **Update Plugin** appears when an update is available.

#### Important Notes

If you know that you have the plugin installed on a child site, but the child site doesn't appear in the list, try to re-sync your sites.

## Monitor Child Sites Live Traffic

To monitor live traffic for one child site from your MainWP Dashboard:

- Open the **Wordfence Overview** tab.
- Locate the child site and click **Live Traffic** in its row action menu.
- The extension opens the live traffic view for that site.

![Screenshot of Live Traffic view showing requests to a child site](/images/add-ons/21f0fedceb53.png)

If live traffic is disabled, open the network-level **Live Traffic** tab, set **Enable live traffic logging** to **All Traffic**, and click **Save Settings**. If the site uses individual overrides, configure the same option from the site's **Wordfence > Live traffic** tab instead.

## Monitor Network Live Traffic

Open the **Live Traffic** tab on the Wordfence page to view traffic from active child sites in one place.
