The MainWP Early Access Extension lets you opt in to pre-release MainWP Dashboard and MainWP Child updates and preview supported MainWP Lab features before they are publicly released. This allows you to test new functionality, provide feedback, and prepare for upcoming changes.

## Key Features

- Enable pre-release update checks for MainWP Dashboard
- Enable pre-release update checks for MainWP Child globally or per site
- Roll MainWP Dashboard or MainWP Child back to the latest stable WordPress.org release
- Preview MainWP Lab features such as Telemetry, MainWP Bridge, MCP with Abilities API, Patchstack Integration, and the new Help Documentation
- Provide early feedback to shape development
- Prepare your workflow for upcoming changes

## Installation

The MainWP Early Access Extension needs to be installed only on your MainWP Dashboard site. You can [install it as any other MainWP Extension](/dashboard/overview/manage-extensions#install-extensions).

## Getting Started

<Steps>
<Step title="Install and activate the extension">
Navigate to **MainWP > Add-ons > Manage Add-ons** and install the Early Access extension.
</Step>
<Step title="Open the Lab page">
Go to **MainWP > Lab** and use the **Early Access Settings** card to open the Early Access settings page.
</Step>
<Step title="Configure update access">
Use **Enable Early Access for MainWP Dashboard** and **Enable Early Access for MainWP Child Plugin** to control pre-release update checks. Add a GitHub Personal Access Token (PAT) if you need a higher GitHub API limit for update checks.
</Step>
<Step title="Manage MainWP Child access per site">
Open a child site's Early Access settings to use **MainWP Child Early Access** with **Use global setting**, **Enable**, or **Disable**.
</Step>
<Step title="Test Lab features">
Use the **Lab** tab to review available feature cards and enable supported options, such as Telemetry.
</Step>
</Steps>

<Warning>
Early access features may be unstable or incomplete. Use them in testing environments before deploying to production sites.
</Warning>
