The Admin Passwords feature changes the password for the connected administrator account on selected Child Sites.

## What This Page Does

- Shows the connected admin user for each site
- Lets you select sites, tags, or clients
- Changes the connected administrator account password
- Shows per-site success or error results

## Update Admin Passwords
<Steps>
<Step title="Open Admin Passwords">
Go to **MainWP > Users > Admin Passwords**.

![Admin Passwords page in MainWP Dashboard](/images/sites/admin-passwords-page.png)
</Step>
<Step title="Select targets">
Choose the Child Sites, Tags, or Clients where you want to update passwords.
</Step>
<Step title="Enter a new password">
In **Change Admin Password**, set the password you want to apply.
</Step>
<Step title="Update passwords">
Click **Update Password**.
</Step>
<Step title="Review results">
Check the results modal for success or error on each selected site.
</Step>
</Steps>

---

## Validation Rules

- You must select at least one site, tag, or client
- You must provide a password
- Suspended sites and sites with sync errors are skipped

---

## Important Notes

<Warning>
This does not update every administrator account on a site. It updates only the connected administrator account password on selected Child Sites.
</Warning>

MainWP communication with child sites continues to use the normal MainWP secure protocol. This page updates WordPress user account passwords.
